# 需求分析设计文档：今日报告 + 每日新词上限 + 测试模块 + 复习功能

> 日期：2026-07-22
> 阶段：Brainstorming → 设计文档（尚未写功能代码）
> 分支：`feature/daily-report-quiz`（基于 `main` @ 63e0e1d）
> 关联：单词本 + 云端账户（`feature/wordbook-account` 已上线 Phase B）

---

## 0. 目标与边界

把"学习—检测—复习"闭环补全：

1. **修 StudyLog 断点 + 今日报告**：让"学员当天掌握情况"可记录、可查看。
2. **每日新词上限**：每用户全局设一个每日新词目标，新词按词本顺序出。
3. **每日测试模块**：单词默写 / 选择题 / 词组默写，三种题型，可选单词范围。
4. **测试结果反馈进掌握评分**：测试对=Good、错=Again，真正驱动 SM-2。
5. **复习功能**：选定"最近 N 天学过的词"，翻卡片快速过。

**不做（YAGNI）**：不引入自适应题库、不引入错题本导出、不做社交/排行榜、不引入听写语音识别（词组默写先用文本输入+长度提示）。

## 1. 需求 1：修 StudyLog 断点 + 轻量「今日报告」

### 现状（已核实）
- `app/(tabs)/index.tsx` 的 `handleGrade` 只调 `reviewWord(repo,...)` 写进度，**漏调 `postStudyLogs`**。
- 后果：云端 `study_logs` 表为空 → 后端 `StatsView.today_count` 与基于日志的 `streak` 恒为 0。
- 前端 `stats.tsx` 只有累计维度（总词数/待复习/正确率），无"今日"维度。

### 修复
- `handleGrade` 评分后调用 `postStudyLogs([{wordbookId, wordId, grade, ts}])`。
  - 本地模式：`postStudyLogs` 经 `repo` 写入（需给 Repository 增加 studylog 读写，或本地用 AsyncStorage 兜底；见 §4）。
  - 云端模式：`httpRepo.postStudyLogs` 已存在，直接接通。
- 为区分"新词首次学"与"复习"，`StudyLog.grade` 已够；是否加 `is_new` 字段见 §4 决策。

### 今日报告（轻量）
在「统计」页或新增「今日」区块展示：
- **今日学习数**：今天 studylog 条数（去重到词）。
- **今日掌握率**：今天 Good/Easy 占比（Again/Hard 计未掌握）。
- **今日词明细**：今天每个词的评级 + 时间，点开可看释义。

数据来源：后端加 `GET /stats/?scope=today`（或前端用 `study-logs` 拉今日后算）。本地模式用 `getWordbookStats` 扩展一个 `today` 视图。

## 2. 需求 2：每日新词上限（每用户全局）

### 已确认决策
- **粒度**：每用户全局一个上限（如 20 个/天），所有词本共用。
- **出新词顺序**：已天然支持——`getNextQuizWord` 的新词按 `WordbookWord` 顺序（id 顺序=词本顺序）出，无需改排序逻辑。
- **达上限行为（默认）**：今日新词已达上限后，**只推到期复习词，不再推新词**（学习不中断，复习继续）。

### 需要的机制
- **每日上限设置**：每用户一个整数 `dailyNewWordGoal`。
  - 本地：`AsyncStorage` key 按 user（`wb_daily_goal_{userId}`）。
  - 云端：后端加 `UserSettings(user_id, daily_new_word_goal)` 表（词汇后端不建 users 表，仅存设置）。
- **今日已学新词计数**：实时知道"今天已学几个新词"。
  - 推荐：基于 `StudyLog` 加 `is_new` 布尔（首次学该词记 true），今日 `is_new=true` 计数即今日新词数。需后端 schema 加字段（migrate）。
  - 本地兜底：用 `progress.lastReviewTs` 今日 + 此前未学过（repetitions 由 0→1）近似，或本地 studylog 计数。
- **选词闸门**：`getNextQuizWord` 在"出新词"分支前判断 `todayNewCount >= dailyNewWordGoal`，满足则跳过新词分支，只走到期复习。

### 设置入口（默认方案）
「我的」页加"每日新词目标"输入框（全局）。

## 3. 需求 3：每日测试模块

### 入口（默认方案）
新增独立 Tab「测试」（与学习/词本/统计/薄弱词/我的并列）。点进后有三个题型卡片 + 范围选择。

### 三种题型
1. **单词默写**（看释义写单词）
   - 显示 translation，用户输入英文单词，忽略大小写/首尾空格比对 `word`。
   - 可选"显示首字母提示"。
2. **单词选择题**（看单词四选一选释义）
   - 显示 `word`，4 个候选 translation（1 正确 + 3 干扰）。干扰词从同词本随机取（不含答案）。
3. **单词词组默写**（给释义默写词组，给定每个单词长度）
   - 显示词组释义（phrase meaning），用户输入整组词组；UI 按 `phrases[].phrase` 拆词给出每个单词的 **长度下划线提示**（如 `_ _ _ _` 表示 4 字母词）。
   - 词组数据来自 `Word.phrases`（JSON：[{phrase, meaning}]）。空则降级提示"该词无词组"。

### 单词范围（可选，全部题型通用）
- ① 当前词本全部
- ② 薄弱词（错率≥34% 或 EF<1.8）
- ③ 最近 N 天学过的词（与需求 5 同口径）
- ④ 自选（从词本挑若干词）
- 默认进入用"当前词本全部"；每日场景用户选"今天学的词"即是对当天所学自测。

### 测试流程
选范围 → 选题型（可多选，依次进行）→ 逐题作答 → 结束给本次测试正确率 + 逐题对错回顾 → **结果写回掌握评分（见 §4）** + 写 studylog（标记来源 `source='quiz'`）。

## 4. 需求 4：测试结果反馈进掌握评分（核心映射）

### 已确认映射规则
| 测试作答 | 映射到 SM-2 grade | 效果 |
|---|---|---|
| 正确（默写对 / 选择对） | **Good (2)** | 推进间隔、记 correct、possible repetition+1 |
| 错误（默写错 / 选择错） | **Again (0)** | 重置间隔、记 wrong、repetition 归零 |

- **词组默写**逐词判定：整组对才算对；任一词错即整组错（或逐词给分，默认整组对/错）。
- 映射通过复用现有 `reviewWord(repo, user, wordbook, word, grade, now)` 完成——测试模块不另写进度逻辑，只调用它。
- 同时写 `StudyLog`（grade + `source='quiz'`），使今日报告/统计一致。

### 数据模型变更（为支持"今日/测试来源"）
- `StudyLog` 加字段 `source`（`study`/`quiz`/`review`，默认 `study`）。
- `StudyLog` 加 `is_new` 布尔（首学标记，供每日新词计数）。
- 两处都需后端 migrate；本地模式在 AsyncStorage 结构里加同名字段。
- **每日上限设置存储**：云端加 `UserSettings` 表（见 §2）。

> 注：`source`/`is_new` 非功能必需（每日新词计数可近似），但若要做准，建议加。Phase 2 plan 再定是否首批 migrate。

## 5. 需求 5：复习功能（最近 N 天学过的词，翻卡）

### 已确认决策
- **周期**：最近 N 天（N 可选 7/14/30，默认 7）内学过的词。
- **形式**：翻卡片快速过（与学习模块同款 `FlashCard` + 四档评分）。

### 实现
- 取数：`progress` 中 `lastReviewTs` 落在 `[now - N*day, now]` 的词（按 user×词本）。
- 翻卡评分：复用 `reviewWord` + 四档（Again/Hard/Good/Easy），写 studylog（`source='review'`）。
- 与学习模块区别：复习不引入新词、不限每日上限，纯"快速过一遍近期接触的词"。

### 入口（默认方案）
新增独立 Tab「复习」，进入先选 N（7/14/30），再翻卡。

> 或合并「测试」「复习」为一个「练习」Tab（内含两个区块）。**待用户确认入口形态**（见 §8）。
