# 设计文档：薄弱词 / 错题本（基于 SM-2 自动算）

- 日期：2026-07-21
- 状态：已确认（brainstorming 通过）
- 关联：wordhoard 改造项目（SM-2 已落地）

## 目标

自动识别并集中复习掌握不牢的词，无需手动标记。基于现有 SM-2 字段（`ef` / `correct` / `wrong`）实时计算，零额外埋点。

## 判定规则（阈值可配置）

```
weak = (wrong > 0 && wrong / (correct + wrong) >= 0.34) || ef < 1.8
```

常量抽到 `lib/weakWords.ts` 顶部，便于调参。

## 数据

- v1：完全复用现有 `Word` 字段，**不新增存储**，列表从 DB 实时计算。
- v1 可选增强（错题历史）：新增 `mistakes` 表 `(id, wordId, ts)`，记录每次答错时间；本版先不做，留作后续。

## UI

- 新增 Tab「薄弱词」（`app/(tabs)/weak.tsx`）：
  - 列表按 `(wrong 降序, ef 升序)` 排序。
  - 每条显示：单词、中文释义、错次徽标（如 `错 5`）、`ef` 小字。
  - 「重练」按钮：把这些词 `id` 注入当前测验队列（`getQuizWord` 优先返回它们）。
  - 列表为空时显示空状态文案。
- 底部 Tab 栏注册该页面（参考 stats Tab 的做法）。

## 数据流

- `lib/weakWords.ts` 导出 `getWeakWords()`：从 DB 取全部词 → 按规则过滤 → 排序 → 返回。
- 重练：写入一个内存/持久化的「优先队列」集合，`getQuizWord` 优先弹出。

## 错误处理

- `correct + wrong === 0` 的词不参与判定（避免除零，且不算薄弱）。
- 词库为空时列表显示引导去背词。

## 测试

- 单测 `getWeakWords()`：给定样例 Word 行（覆盖阈值边界：刚过线 / 不过线 / ef 边界 / 无答题记录），断言分类正确。

## 验收标准

- 连续答错几个词后，切到「薄弱词」Tab 能看到它们并按错次排序。
- 点「重练」后，首页测验优先出现这些词。
